## 用户访谈新增发现（2026-04-17）

来源：与目标用户（B2B SaaS 创始人，个人 IP 抖音账号运营者）的情境访谈。

### 发现 1：目标用户比 PRD 预设更高级，痛点在上游
PRD 假设用户卡在内容生产（写不出来）。实际上，部分用户已有完整半自动化生产管线（研究爆款 → Prompt 框架 → Claude 生成脚本 → 分镜 → TTS → 视频渲染），生产效率已基本解决。
**真正的瓶颈在选题洞察**：不知道该生产什么内容，无法判断为什么某内容是热点，以及热点背后的情绪触发机制是什么。

**产品影响**：需要在 PRD 中补充"选题洞察模块"——爆款拆解 + 情绪触发分析 + 选题推荐。这在当前 PRD 中几乎没有覆盖。

### 发现 2：TTS 与分镜不同步 + 目标时长 60 秒
用户目标：视频控制在 60 秒内。当前生成约 4 分钟 TTS + 40 帧分镜，AI 渲染无法对齐。

**根本原因**：60 秒 / 40 帧 = 每帧 1.5 秒，无法承载逐帧旁白。架构选型错误。

**正确架构（60秒视频）**：
- 脚本：连续旁白，总字数 180-210 字（200字/分钟 × 60秒）
- 分镜：15-18 帧（每帧 3-4 秒），与旁白并行而非 1:1 对应
- 节奏分布：开场 3 帧快切（2秒/帧）→ 中间 8 帧正常（4秒/帧）→ 结尾 4 帧减速（5秒/帧）
- 脚本结构：开场钩子（10秒/35字）→ 核心论点（15秒/50字）→ 单一案例（25秒/80字）→ 金句收尾（10秒/35字）

**产品影响**：
- 内容生产管线需区分"长视频模式"（40帧，4分钟）和"短视频模式"（15帧，60秒）
- 60 秒模式脚本上限硬控制 210 字，超出则提示截断
- 分镜帧数根据时长自动推算（用户输入目标时长，系统输出对应帧数）

### 发现 3：存在两种核心内容公式，选题难度不同
- **公式一（挑衅断言型）**：反常识开场 → 痛感词汇 → 案例证明 → 真相揭示。生产门槛低，格式可复制。
- **公式二（日常现象洞察型）**：生活中习以为常的现象 → 揭示经济学/心理学/商业本质 → 认知升级。例："为什么牛奶盒方的可乐瓶圆的"、"为什么淘宝卖衣服出现预售模式"。
  - 这种公式内容质量 70% 取决于选题，30% 取决于写法。

**产品影响**：内容策略模块需支持多种内容公式选择；选题推荐功能对公式二用户价值极高。
