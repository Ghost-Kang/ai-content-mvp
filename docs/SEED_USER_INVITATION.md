## 种子用户选择标准（硬线）

满足 **4 选 3** 才发邀请：

- [ ] 现在在做 B2B SaaS（不是纯 To C、不是外包公司）
- [ ] 团队 ≤ 30 人（这是目标画像，大公司用不上）
- [ ] 最近 3 个月真的发过抖音 / 小红书（不是"打算做"）
- [ ] 愿意回复 ≥ 1 次具体反馈（不是"挺好的"一句话打发）

**不要邀请**：
- 朋友客气型 —— 会给虚假正反馈
- 外包 / 代运营公司 —— 他们的需求是另一个产品
- 大厂同学 —— 用完即走，反馈空洞

## 邀请文案模板（微信）

```
Hi [名字]，

做了个小工具：输入产品 + 受众 + 核心主张，30 秒出一条 60 秒抖音脚本，17 帧分镜 + 评论引导问题，

还在内测，想找 3 个 B2B SaaS 创始人试用一周，给我 1-2 条具体反馈。不需要天天用，
生成过 3-5 条脚本就够。

有兴趣吗？链接：https://<prod-url>/
注册用任意邮箱就行（Clerk 一键登录），我这边会手动把你的配额上调。

—— [你的名字]
```

**为什么这么写（别改）**：
- 一句话说清是什么 + 给谁 —— 直接筛掉不符合的
- "30 秒"是记忆点，不是"AI 生成"
- "1-2 条具体反馈"降低心理门槛 —— 不是请求写 10 页评估
- 不提"beta、MVP、测试" —— 这些词暗示质量低

## 反馈收集

**不要在 app 里做 in-app 反馈表**（对 MVP 太重）。**用外部表单**：

1. 创建腾讯问卷 / 飞书多维表单，5 道题（≤ 5 分钟填完）：
   - 1. 你用这个工具解决的真实场景是什么？（open）
   - 2. 生成的脚本你有多少能直接用？0% / 25% / 50% / 75% / 90%+
   - 3. 最想改动的 1 件事？（open）
   - 4. 相比现在的做法（自己写 / 让同事写 / 外包），快了多少？
   - 5. 如果收费，你觉得 ¥[X]/月 合理？（给三档：¥19 / ¥49 / ¥99 / 其他）
2. 把表单链接放在邀请文案第二封（跟进时发）
3. **不要**把反馈链接加到 landing page —— 种子用户才该看到

## 绝对不要做
